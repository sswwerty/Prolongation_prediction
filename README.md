# Прогноз пролонгации страховых договоров

## Описание задачи

В рамках тестового задания необходимо построить модель, предсказывающую вероятность пролонгации страхового договора автомобиля.

Целевая переменная:
- `1` — договор будет продлён  
- `0` — договор не будет продлён  
- `-1` — требуется предсказать  

---

## Особенность постановки задачи

В рамках задания не были заданы конкретные бизнес-требования к модели:
- не указаны ограничения на интерпретируемость
- отсутствуют регуляторные требования
- не задан приоритет между метриками
- не определён формат использования модели (скоринг, фильтрация, объяснение решений и т.д.).

Поэтому я выбрала рассмотреть задачу не как построить конкретную модель и оптимизировать по конкретной метрике, а исследовать несколько возможных подходов, которые могут возникнуть ввиду каких-то дополнительных ограничений или требований, которые могут быть.

---

## Подход к решению

Был реализован полный ML pipeline:

- аудит данных (пропуски, типы, дубликаты);
- исследовательский анализ (EDA)
- разбиение на train / validation / test
- feature engineering
- обучение нескольких моделей
- сравнение по validation-выборке
- подбор threshold
- калибровка вероятностей
- финальная оценка на test
- построение предсказаний для score-выборки

---

## Рассмотренные модели

В рамках исследования были обучены три модели с разными свойствами:

- **Logistic Regression** : базовая `интерпретируемая` модель  

- **Random Forest**: `устойчивый` ансамбль с хорошим качеством  

- **CatBoost**: модель, ориентированная на табличные данные с категориальными признаками и дающая `высокие показатели`  

Такой выбор позволяет посмотреть на задачу с разных сторон:  
интерпретируемость, стабильность, максимальное качество.

---

## Метрики

Использовался набор метрик:

- **PR-AUC** — основная метрика (качество по положительному классу)
- **ROC-AUC** — качество ранжирования
- **F1-score** — баланс precision/recall
- **Brier score** — качество вероятностных оценок

---

## Логика выбора модели

 Выбор модели выполнялся **на основе метрик на validation-выборке**.

Это осознанное решение:
- при отсутствии внешних требований метрики становятся основным критерием
- test-выборка не использовалась для выбора модели
- сравнение проводилось по совокупности метрик, а не по одной

При этом важно:

- Random Forest показывает немного более высокое значение PR-AUC
- CatBoost превосходит его по ROC-AUC, F1 и Brier score
- Logistic Regression ожидаемо уступает по качеству, но остаётся интерпретируемой

В рамках данного решения выбрана **CatBoost** как модель с наиболее стабильным качеством по совокупности метрик.

---

## Интерпретация выбора

Важно подчеркнуть, что выбор модели в данном проекте:

- не является универсально “лучшим”  
- является оптимальным

Если бы требования задачи были другими:
- при жёстких требованиях к интерпретируемости → была бы выбрана Logistic Regression
- при ограничениях на сложность → мог бы быть выбран Random Forest
- при акценте на качество → CatBoost остаётся предпочтительным
