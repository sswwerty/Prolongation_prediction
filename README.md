# Прогноз пролонгации страховых договоров

## Описание задачи

В рамках тестового задания необходимо построить модель, предсказывающую вероятность пролонгации страхового договора автомобиля.

Целевая переменная:
- `1` — договор будет продлён  
- `0` — договор не будет продлён  
- `-1` — требуется предсказать  

---

## Особенности постановки задачи

В задание не было указано дополнительных бизнес-требований к модели (например, ограничения на интерпретируемость, регуляторные требования или приоритет конкретной метрики).

Поэтому я рассмотрела задачу в общем виде: как построение и сравнение нескольких подходов к решению с целью понять, как они ведут себя на данных и какие компромиссы дают.

Такой подход позволяет продемонстрировать разные варианты решения, которые могут быть релевантны в зависимости от дальнейшего контекста использования модели.

---

## Подход к решению

Был реализован полный ML pipeline:

- аудит данных (пропуски, типы, дубликаты)
- исследовательский анализ (EDA)
- разбиение на train / validation / test
- feature engineering
- обучение нескольких моделей
- сравнение по validation-выборке
- подбор threshold
- калибровка вероятностей
- финальная оценка на test
- построение предсказаний для score-выборки

---

## Рассмотренные модели

В рамках работы были обучены модели с разными свойствами:

- **Logistic Regression** — базовая интерпретируемая модель  
- **Random Forest** — устойчивый ансамбль с хорошим качеством  
- **CatBoost** — модель, ориентированная на табличные данные с категориальными признаками  

Такой набор моделей позволяет рассмотреть задачу с разных сторон:  
интерпретируемость, устойчивость и максимальное качество.

---

## Метрики

Использовался набор метрик:

- **PR-AUC** — основная метрика (качество по положительному классу)
- **ROC-AUC** — качество ранжирования
- **F1-score** — баланс precision/recall
- **Brier score** — качество вероятностных оценок

---

## Логика выбора модели

Выбор модели выполнялся на основе качества на validation-выборке.

При этом:
- test-выборка не использовалась для принятия решений
- сравнение проводилось по совокупности метрик, а не по одной

По результатам:

- Random Forest показывает немного более высокое значение PR-AUC
- CatBoost превосходит его по ROC-AUC, F1 и Brier score
- Logistic Regression уступает по качеству, но остаётся интерпретируемой

В рамках данного решения выбрана **CatBoost** как модель с наиболее стабильным качеством по совокупности метрик.

---

## Интерпретация выбора

Выбор модели зависит от целей использования:

- при необходимости высокой интерпретируемости может быть предпочтительна Logistic Regression
- при акценте на простоту и устойчивость — Random Forest
- при приоритете качества и точности вероятностных оценок — CatBoost

Таким образом, выбор модели определяется не только метриками, но и контекстом задачи.
